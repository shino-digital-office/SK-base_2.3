# SK-base v2.2 | cardカタログ見本

本ページ(examples/cat-card.html)は、ポートフォリオや案件制作の際に使える **cardのカタログ** です。

単なる見た目サンプルではなく、

- **カード単体の構造設計**
- **カード配置（グリッド）の設計**
- **背景・画像・テキストの役割分離**

を前提に、「考えなくてよい部分」を最大限減らすことを目的としています。

---

## このキットでできること

- よく使うカード UI を **型として即コピペ**
- **card（中身）** と **cards（配置）** を分離した設計
- 画像を
  - 「意味を持つ画像（img）」
  - 「意味を持たない背景（bg / span）」  
    に明確に分けて実装
- カードのサイズ変更・レイアウト変更は **CSS 調整のみ**
- UI 型 / 紙面型どちらにも転ばせられる拡張性

---

## 収録カードタイプ

### Card（カード単体）

| クラス名      | 特徴                                                     |
| ------------- | -------------------------------------------------------- |
| `card--cover` | 背景画像がカード全体を覆うタイプ（背景は意味を持たない） |
| `card--stack` | 画像＋テキストが上下に積まれる基本カード                 |
| `card--split` | 画像＋テキストが左右に分かれる UI 型カード               |

### Cards（カード配置）

| クラス名            | 特徴                                 |
| ------------------- | ------------------------------------ |
| `cards`             | 基本の自動グリッド                   |
| `cards--col`        | 縦並び                               |
| `cards--mixed`      | 上段 2 列＋下段 3 列の混在レイアウト |
| `cards--last-child` | 最後のカードだけ中央寄せ             |

## 設計思想

### 1. Card と Cards を分離する

- **card**  
  → 見た目・中身・サイズを決める責務
- **cards**  
  → 並べ方・余白・列数を決める責務

```scss
.cards {
  display: grid;
  gap: var(--cards-gap, 20px);
  justify-items: center;
}
```

カードの max-width を変えたい場合でも、
配置ロジックを書き換えずに済むようにしています。

### 2. 画像の「意味」で実装を分ける

背景として使う画像（意味を持たない）

- span.card\_\_bg
- background-image
- object-fit や aspect-ratio はカード側で管理

👉 card--cover で採用

コンテンツとして使う画像（意味を持つ）

- <img>
- alt を持たせる
- レイアウトの一部として扱う

👉 card--stack / card--split で採用

### 3. サイズは「カード自身」が決める

```scss
.card--stack {
  width: 100%;
  max-width: 335px;
}
```

- カードは 自分の最大幅を知っている
- グリッドは それを並べるだけ

cards--mixed や cards--last-child では
必要に応じて max-width を解除し、配置側に委ねています。

### 4. transform の使用ルール

transform は以下に限定して使用します。

- 背景レイヤー
- 疑似要素
- 装飾専用レイヤー

❌ レイアウトの骨組み・位置決定には使用しません
（grid / flex / inset を優先）

## 基本的な使い方

1. index.html でカード型・配置型を確認
2. 必要な card と cards をコピー
3. SCSS をプロジェクトに取り込む
4. max-width / gap / カラム数を調整

HTML 構造は基本的に 型を崩さず使うことを想定しています。

## cards--mixed について

```scss
.cards--mixed {
  grid-template-columns: repeat(6, 1fr);
}
```

- 6 分割グリッドを基準に
  - 上段：3 カラム × 2 枚
  - 下段：2 カラム × 3 枚

- カード幅を変えたい場合は
  グリッド分割数と占有数をセットで調整

👉 「カードサイズ」と「配置設計」は常に連動します。

## 今後の拡張方針

- 新しい card 型 を追加しても、cards 側は流用可能
- 新しい cards 配置 を追加しても、card 型は流用可能
- このリポジトリを母艦として、案件ごとに必要な型だけを切り出す運用を想定
